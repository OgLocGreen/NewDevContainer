# Scratch — [Project Name]

> Project-relevant side notes, half-formed ideas, and observations that don't (yet) fit a structured document.
> One Scratch folder per parent project (not per subproject).
> Append freely. Promote to a structured document when an idea matures. Discard when irrelevant.

## How to use

- **Append-friendly:** add new entries on top with a date stamp
- **No format requirements:** prose, bullets, sketches — whatever captures the thought
- **Promote or discard:** when a scratch entry develops into something concrete, move it to the right document via `/push`. When it stops being useful, delete it (this is the only vault content where deletion is fine)
- **Cross-link:** when a scratch entry relates to a paper, AP, or decision, link it via `[[...]]`
- **Tag by subproject:** if hierarchical, tag entries by subproject (e.g. `[Paper 1]`, `[Paper 2]`) so they stay findable
